# Contributing

Thanks for your interest in contributing.

## Scope

This repository is primarily a technical case study plus synthetic benchmark data.

## How To Contribute

- Open an issue describing the change before large pull requests.
- Keep changes focused and small.
- Preserve synthetic-only data requirements; do not introduce live or customer-identifiable records.
- Update README documentation when behavior or structure changes.

## Pull Request Checklist

- Explain the problem and proposed fix clearly.
- Include before/after behavior when applicable.
- Ensure links and referenced files exist.
- Run the demo locally:

```bash
python3 clean_waveform_benchmark/inference_demo.py
```

## Code Style

- Prefer readable, explicit code over clever shortcuts.
- Avoid adding dependencies unless necessary.
